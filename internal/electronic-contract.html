자체 개발 검토

3. 법적 측면 — 자체 개발이 합법인가

2020-12-10 전자서명법 개정이 자체 개발 가능성을 열었다

핵심 사실

2020년 12월 10일 전자서명법 개정으로 공인전자서명 제도가 폐지됐다. 이전엔 공인인증서만 법적 추정력이 있었지만 개정 후엔 모든 인증사업자가 발행하는 전자서명이 동등한 효력을 가진다.

일반 전자서명도 당사자 간 약정이 있으면 서명·기명날인으로서의 효력을 인정받는다. 단 분쟁이 생겼을 때 진정성·무결성 입증 책임은 본인 부담이다.

결론: 자체 개발한 일반 전자서명도 모두사인과 같은 법적 효력을 가질 수 있다. 단 분쟁 대비 감사 추적·해시 무결성 설계는 필수다.

6. 비용 추산 — 손익분기는 언제인가

초기 투자 vs 건당 절감

초기 개발 (MVP)

1팀 컨텍스트 = Claude Code 자체 개발. 비용 절감이 자체 개발 결정의 본질이므로 외주 가정 자체가 부적합.

· 사용자 시간 = 약 1~2주 (Claude Code 페어 개발)

· 인프라·도메인·SSL 셋업 = 거의 0 (Vercel·Supabase 무료 티어)

· 법무 자문 = K-Startup 원스톱 지원센터 무료 법무 상담 활용 (백민혁 팀원 공유, 2026-05-29 오전 11:29)

· 합계 = 사실상 0원 (1팀 시간 외 외부 비용 없음)

운영 변동비 (건당)

본인 인증 약 40원 + 카카오 알림톡 약 10원 + 인프라 약 5원 = 약 55원/건

모두사인 800원 - 자체 55원 = 건당 약 745원 절감

손익분기 (Claude Code 자체 개발 + K-Startup 무료 법무): 초기 투자 사실상 0 → 운영 시작 시점부터 즉시 절감 시작
· 운영 초기 (각 1,000명, 연 9,500건) → 연 약 700만 원 절감
· 1만 명 시점 (연 9.4만 건) → 연 약 7,000만 원 절감
본질: 자체 개발은 비용 절감 결정. Claude Code 페어 개발 + K-Startup 원스톱 무료 법무 상담이면 외부 비용 사실상 0. 외주 가정으로 BEP를 늦추는 분석은 모모케어 컨텍스트와 맞지 않음.

9. 본 단계 결정 (1팀 컨펌 필요)

발표·재무 시뮬레이션·BMC 반영

1팀 결정안

· 가설 검증·발표 단계 = 모두사인 단기 사용 (1~2개월, 검증 끝나면 즉시 자체 개발)

· 자체 개발 진입 시점 = 운영 진입 직전 (Claude Code 1~2주, 초기 비용 거의 0)

· BMC·재무 시뮬레이션 반영 문구 = "전자계약은 자체 개발, 운영 변동비 약 55원/건 (모두사인 대비 745원 절감)"

⚠ 본 단계 사용량·BEP 수치는 모두 가정 기반 시안. 1팀 검토 거쳐 갈음 예정.

10. 운영단 이관 액션

사업화 후 사이클

1. 본인 인증 사업자 1차 선정 (KG이니시스 통합인증 또는 카카오 인증 SDK) + 사업자 등록 후 정식 단가 협상

2. Claude Code 페어 개발 — 사용자 1~2주 사이클로 MVP 작동 (pdf-lib · @signpdf/signpdf · Vercel Blob · Supabase append-only)

3. 감사 추적·해시 무결성 설계 직접 검증 (분쟁 대비)

4. 법무 1차 검토 = 1팀 자체 + K-Startup 원스톱 지원센터 무료 상담 (전자서명법·개인정보보호법·근로기준법)

5. 본인 인증 사업자 약관 검수 + 운영 변동비 실측
